# 学習ログ＆バッチ分析アプリ テスト仕様書

**バージョン：** 0.1  
**更新日：** 2026年4月28日  
**対象バージョン：** 要件定義 v0.4 / API仕様 v0.1

---

## 1. 概要

### 1.1 目的

本書は学習ログ＆バッチ分析アプリの品質を担保するためのテスト仕様を定義する。各機能が要件定義・API仕様・画面定義の通りに動作することを確認する。

### 1.2 テスト対象

| レイヤー | 内容 |
|----------|------|
| バックエンド API | Spring Boot + Spring WebFlux の REST エンドポイント |
| バッチ | Spring Batch の日次集計ジョブ |
| フロントエンド | Next.js 画面の表示・操作・バリデーション |
| SSE | リアルタイム通知の配信 |

### 1.3 テスト種別

| 種別 | 目的 | 主なツール |
|------|------|-----------|
| 単体テスト | サービス層・バリデーションロジックの検証 | JUnit 5 / Jest |
| 統合テスト | API エンドポイントと DB の連携検証 | WebTestClient + TestContainers |
| E2E テスト | 画面操作から API 連携までの一気通貫検証 | Playwright |

### 1.4 前提条件

- テスト実行時は PostgreSQL (Docker) が起動済みであること
- テスト用 Google OAuth は Mock / テスト用クレデンシャルを使用すること
- 各テストは独立して実行でき、実行順序に依存しないこと
