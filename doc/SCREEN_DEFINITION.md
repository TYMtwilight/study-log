# 学習ログ＆バッチ分析アプリ 画面定義書

## 1. 目的

本書は `CLAUDE.md` の要件定義をもとに、Next.js フロントエンドの画面構成と項目定義を整理したものです。ユーザー操作の流れと表示内容を明確にし、実装時の設計指針とします。

## 2. 共通仕様

- 認証: Auth.js v5 の Google OAuth によるログイン
- UI フレームワーク: Tailwind CSS
- バリデーション: Zod で入力値検証
- 全ページをログイン必須とする
- 画面は App Router によるページ構成
- レスポンシブ対応: スマートフォンと PC を想定

## 3. 共通コンポーネント

### 3.1 ヘッダー

- ロゴ / アプリ名
- ナビゲーションリンク
  - ダッシュボード
  - 学習ログ一覧
  - 科目管理
  - CSV ダウンロード
  - バッチ履歴
- ユーザー情報表示（アイコン・名前）
- ログアウトボタン

### 3.2 トースト通知

- 成功メッセージ（CRUD 操作完了時: ログ登録・編集・削除、科目登録・編集・削除）
- 失敗メッセージ（操作エラー時）
- バッチ完了通知
- ログ登録完了通知

### 3.3 共通状態

- ローディング
- API エラー
- セッション切れリダイレクト

## 4. 画面一覧

| 画面 | パス | 目的 |
| --- | --- | --- |
| ログイン画面 | `/login` | Google OAuth によるログイン処理と未認証時の入り口 |
| ダッシュボード | `/` | 今日・今週・月次サマリーの可視化 |
| 学習ログ一覧 | `/study-logs` | 学習ログの一覧、フィルタ、編集、削除 |
| 学習ログ登録/編集 | `/study-logs/new`, `/study-logs/[id]/edit` | 学習ログの新規登録と編集 |
| 科目マスター一覧 | `/subjects` | 科目一覧、追加、編集、削除 |
| 科目登録/編集 | `/subjects/new`, `/subjects/[id]/edit` | 科目の追加と編集 |
| 月次サマリー | `/reports/monthly` | 月別サマリー表示と CSV ダウンロード |
| バッチ履歴 | `/batch-history` | バッチ実行結果の確認 |
| 404 | `*` | 存在しないページの案内 |

## 5. 画面詳細

### 5.1 ログイン画面

- 表題: `ログイン`
- 説明: `Google アカウントでログインしてアプリにアクセスします`
- ボタン: `Google でログイン`
- 処理:
  - 認証成功時: `/` へ遷移
  - 認証失敗時: エラーメッセージ表示

### 5.2 ダッシュボード

#### 5.2.1 表示項目

- 今日の学習時間
  - 本日合計の学習時間（分）
- 今週の学習時間
  - 週次の科目別積み上げグラフ
- 月次サマリー
  - 最新バッチ集計結果のグラフまたはカード表示
- リアルタイム更新状態
  - 学習ログ登録時に SSE で今日の合計学習時間をリアルタイム更新

#### 5.2.2 機能

- 学習時間サマリーのカード表示
- グラフ切り替え（本日／今週／月別）
- 学習ログ追加リンク

### 5.3 学習ログ一覧

#### 5.3.1 表示項目

- 日付
- 科目名
- 学習時間（分）
- メモ
- 作成日時

#### 5.3.2 フィルタ/検索

- 学習日の期間絞り込み（開始日〜終了日）
- 科目選択
- キーワード検索（メモ／科目名など）

#### 5.3.3 操作

- 追加ボタン: `新規ログ登録`
- 編集ボタン: 各行に `編集`
- 削除ボタン: 各行に `削除`
- 行選択: クリックで詳細/編集画面へ

#### 5.3.4 バリデーション

- `科目` は必須
- `学習時間` は必須かつ正の整数
- `学習日` は必須

### 5.4 学習ログ登録/編集

#### 5.4.1 入力項目

- 科目: プルダウン
- 学習時間: 数値入力（分）
- 学習日: 日付入力
- メモ: テキストエリア

#### 5.4.2 ボタン

- 保存
- キャンセル

#### 5.4.3 備考

- 編集時は既存値を初期表示
- 登録完了後は一覧またはダッシュボードへ遷移

### 5.5 科目マスター一覧

#### 5.5.1 表示項目

- 科目名
- 登録日
- 操作

#### 5.5.2 操作

- 追加ボタン: `科目を追加`
- 編集: 各行に `編集`
- 削除: 各行に `削除`
  - 紐づく学習ログがある場合は、確認ダイアログを表示して関連ログもまとめて削除する

#### 5.5.3 削除制御

- 削除ボタン押下時に確認ダイアログを表示
  - `科目を削除すると、関連する学習ログもすべて削除されます。よろしいですか？`
- ユーザーが確認した場合のみ削除を実行
- 削除後は一覧を更新し、成功メッセージを表示
- 削除処理に失敗した場合はエラーメッセージを表示

### 5.6 科目登録/編集

#### 5.6.1 入力項目

- 科目名: テキスト入力

#### 5.6.2 ボタン

- 保存
- キャンセル

#### 5.6.3 バリデーション

- `科目名` は必須
- 同一名の重複チェック（任意）

### 5.7 月次サマリー / CSV ダウンロード

#### 5.7.1 表示項目

- 選択月
- 科目別合計学習時間
- 合計学習時間
- グラフ表示

#### 5.7.2 操作

- 月選択
- CSV ダウンロードボタン

### 5.8 バッチ履歴

#### 5.8.1 表示項目

- 実行日時
- ステータス
- 処理件数
- 開始日時
- 終了日時

#### 5.8.2 操作

- `今すぐ実行` ボタン: 手動でバッチジョブを即時実行する
  - 実行中はボタンを非活性にしてローディング表示
  - 完了時（成功/失敗ともに）トースト通知を表示
  - 完了後、一覧を自動更新して最新の実行結果を反映する
- 詳細表示

## 6. 画面遷移フロー

- `/login` -> 認証成功 -> `/`
- `/` -> `/study-logs`
- `/study-logs` -> `/study-logs/new`
- `/study-logs` -> `/study-logs/[id]/edit`
- `/` -> `/subjects`
- `/subjects` -> `/subjects/new`
- `/subjects` -> `/subjects/[id]/edit`
- `/` -> `/reports/monthly`
- `/` -> `/batch-history`

## 7. 追加 Considerations

- バッチ実行完了の通知は画面右上トーストで表示
- SSE 接続状態の表示をダッシュボードに置く（例: 接続中 / 切断）
- モバイル対応は最低限、縦表示で一覧とフォームがスクロール可能
- エラーハンドリングとして 401/403 は `/login` へリダイレクト
